\begin{abstract}
This report presents an implementation of SASRec trained on MovieLens~1M and
evaluated under full-ranking. A controlled ablation varies each of four
architectural dimensions --- hidden size, number of attention blocks, number of
heads, and sequence length --- one at a time against a shared baseline, plus a
combined configuration that sets all dimensions to their larger values. Hidden
size is the most impactful single dimension (+24.2\% validation NDCG@10 over
baseline), while increasing heads alone hurts when the per-head dimensionality
falls to 16. The combined configuration achieves a test Recall@10 of 0.2007 and
NDCG@10 of 0.1032. Neither model triggered early stopping at 50 epochs,
indicating the training budget was the binding constraint.
\end{abstract}

\maketitle


\section{Introduction}

Matrix factorization compresses a user's entire interaction history into a fixed
vector. Sequential recommendation takes a different approach: it treats the
history as an ordered sequence and learns which past items are most predictive
of the next one, making temporal order a first-class feature rather than
something to be averaged away.

SASRec~\cite{kang2018self} addresses next-item prediction with a
Transformer-style architecture~\cite{vaswani2017attention}. Stacked multi-head
self-attention blocks process the interaction sequence under a causal mask, so
each position attends to all preceding items but not future ones. The model is
implemented in PyTorch and evaluated on MovieLens~1M~\cite{harper2015movielens}
using full-ranking NDCG@10, NDCG@20, Recall@10, and Recall@20.


\section{Data Preprocessing}

The MovieLens~1M dataset contains 1,000,209 ratings from 6,040 users on 3,706
movies in the format \texttt{userId::movieId::rating::timestamp}.

Ratings below 4.0 are discarded; only higher-rated interactions are kept as
implicit positive feedback. Each user's retained interactions are sorted by
timestamp to form a chronological item sequence. Users with fewer than five
interactions are removed, since the split below requires at least three items
and very short sequences provide little sequential signal.

User and item IDs are remapped to contiguous integers starting from 1; index 0
is reserved for padding. For a user sequence of length $n$, the first $n-2$
items form the training sequence, item $n-1$ is the validation target, and item
$n$ is the test target. The complete interaction history per user is stored to
exclude already-seen items from the candidate set at evaluation time.

Training samples are constructed token-wise: the input at each non-padding
position is the current item and the target is the next item. Sequences are
left-padded to a fixed length $L$; padding positions contribute nothing to the
training loss.


\section{Model Architecture}

\subsection{Embeddings}

Each item is mapped to an $H$-dimensional vector via a learnable embedding table
of size $(N+1) \times H$, where index 0 is fixed to zero. Position embeddings
of the same dimension are added elementwise:

\begin{align}
  \mathbf{x}_t = \mathbf{e}_{i_t} + \mathbf{p}_t, \quad t = 1, \dots, L.
\end{align}

Padding positions are explicitly zeroed before the first block and after each
subsequent block. Residual connections can restore non-zero values at padding
slots even when the attention mask prevents information flow, so the explicit
zeroing is necessary.

\subsection{Self-Attention Blocks}

The model stacks $B$ identical blocks. Following Xiong et al.~\cite{xiong2020layer},
layer normalisation is applied \emph{before} each sub-layer (pre-LN), which
stabilises gradient flow in deeper networks.

The self-attention sub-layer uses a causal mask: entry $(i,j)$ is blocked when
$j > i$, preventing position $i$ from attending to any future position. A
key-padding mask additionally prevents all positions from attending to padding
tokens. The point-wise feedforward sub-layer is:

\begin{align}
  \text{FFN}(\mathbf{x}) = W_2 \cdot \text{GELU}(W_1 \mathbf{x} + b_1) + b_2,
\end{align}

where the inner dimension is $4H$. GELU~\cite{hendrycks2016gaussian} is used
instead of ReLU; it avoids the hard zero gate, which improves training stability
for attention-based models. Both sub-layers include residual connections and
dropout.

\subsection{Prediction}

After all $B$ blocks a final layer normalisation is applied. The hidden state
at the last sequence position, $\mathbf{h} \in \mathbb{R}^H$, represents the
user's current preference. Candidate items are scored by dot product:

\begin{align}
  \text{score}(j) = \mathbf{h}^\top \mathbf{e}_j.
\end{align}


\section{Training Setup}

\subsection{Objective}

The model is trained with binary cross-entropy and one uniformly sampled
negative per positive. At each non-padding position $t$:

\begin{align}
  \mathcal{L} = -\sum_{t:\, i_{t+1} \neq 0}
    \Bigl[
      \log \sigma(\text{score}(i_{t+1}))
      + \log(1 - \sigma(\text{score}(j_t)))
    \Bigr],
\end{align}

where $j_t$ is drawn uniformly from items the user has not interacted with.

\subsection{Optimisation}

Adam~\cite{kingma2015adam} is used with learning rate $10^{-3}$ and weight
decay $10^{-5}$. Gradient norms are clipped at 5.0. Training runs for up to 50
epochs; if validation NDCG@10 does not improve for five consecutive epochs,
training stops early and the best checkpoint is used for test evaluation.

\section{Evaluation and Results}

\subsection{Protocol}

Evaluation uses full ranking: the model scores all items not previously seen by
each user, and the held-out target is ranked within that set. Items in the
user's full history are masked with $-\infty$ before ranking, except for the
target itself, so the model is not penalised for assigning low scores to items
it was never asked to predict.

Recall@$K$ is 1 if the target appears in the top $K$ positions, else 0.
NDCG@$K$ additionally discounts by rank:

\begin{align}
  \text{NDCG@}K =
  \begin{cases}
    \dfrac{1}{\log_2(\text{rank} + 2)} & \text{if rank} < K, \\
    0 & \text{otherwise.}
  \end{cases}
\end{align}

Sampled evaluation has been shown to produce rankings that differ from full
evaluation and should not be used for final
reporting~\cite{krichene2020sampled}, so full ranking is used throughout.

\subsection{Per-Dimension Analysis}

\textbf{Hidden size} is the most impactful single change. Doubling $H$ from 64
to 128 improves validation NDCG@10 by 24.2\% and test NDCG@10 by 15.1\%. Wider
embeddings give the model more capacity to differentiate items and the FFN
layers scale as $4H$, so the benefit compounds.

\textbf{Sequence length} is the second largest contributor at +8.8\% on
validation NDCG@10. Extending $L$ from 50 to 100 allows the model to condition
on more of each user's history. Many MovieLens~1M users have more than 50
positive ratings, so the additional context is genuinely useful.

\textbf{Number of blocks} provides a modest but consistent gain of +2.2\%.
Adding a third attention block allows one more layer of non-linear
transformation, but the improvement is small relative to hidden size, suggesting
that two blocks already capture most inter-item dependencies at this dataset
scale.

\textbf{Number of heads} is the only dimension where increasing the value hurts
performance, dropping validation NDCG@10 by 4.1\%. At $H = 64$ with four heads,
each head operates on a 16-dimensional subspace --- too narrow to learn
meaningful attention patterns. The extra projection matrices add parameters
without useful capacity. This is also the only configuration to trigger early
stopping, at epoch 49. When heads are increased in \texttt{config\_deeper},
$H$ is simultaneously doubled to 128, giving each head 32 dimensions, which is
sufficient to recover the benefit.

\textbf{Combined configuration.} \texttt{config\_deeper} outperforms the
baseline by 28.5\% on validation NDCG@10, larger than any single-dimension
gain. The head-dimension interaction explains part of this: additional heads
only help when $H$ is large enough to support them. The remaining uplift likely
reflects non-additive interactions between depth, width, and context length that
the single-dimension ablations cannot capture individually.


\section{Discussion}

The implementation uses pre-layer normalisation, causal masking, $4\times$ FFN
expansion, and GELU activations. Full-ranking evaluation is used throughout to
avoid the biases introduced by sampled evaluation.

The ablation yields two results that a two-configuration comparison would miss.
Hidden size dominates the other three dimensions at this scale, and increasing
heads without widening the model makes things worse. Both findings have direct
practical implications: on a fixed compute budget, widening the model is a
safer investment than deepening it or adding heads.

Three improvements stand out. First, training longer: five of six configurations
were still improving at epoch 50, and a cosine learning rate schedule with a
longer budget would likely push all numbers up. Second, harder negatives:
uniform sampling means the model rarely sees items close to the decision
boundary; popularity-weighted or in-batch negatives would provide sharper
gradients. Third, a joint sweep over $H$ and the number of heads keeping
$H / \text{heads} \geq 32$ would give a cleaner picture of the head-size
interaction than the current independent ablation.


\section{Conclusion}

Six SASRec configurations are compared on MovieLens~1M under full ranking. The
ablation shows that hidden size is the most valuable single dimension to scale,
while increasing heads without a corresponding increase in hidden dimension
actively hurts performance. The combined configuration achieves Recall@10 of
0.2007 and NDCG@10 of 0.1032. No configuration converged within 50 epochs, so
the training budget rather than model capacity appears to be the current
bottleneck.
